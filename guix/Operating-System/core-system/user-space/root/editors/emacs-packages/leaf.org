** Overview
:PROPERTIES:
:tangle: no
:END:

Leaf is a declarative configuration framework for Emacs that provides a clean, macro-based interface for managing packages and their configuration. It's the foundation of the literate-config-system.

*Key Features:*
- Declarative package configuration syntax
- Automatic dependency ordering
- Keyword-based configuration (like use-package but more flexible)
- No external package managers needed (we use Guix)
- Clean, readable configuration blocks

** Description
:PROPERTIES:
:tangle: no
:END:

Leaf provides a unified way to configure Emacs packages using a ~leaf~ macro. Instead of manually calling ~require~, setting variables, and adding hooks, you declare everything in one form:

#+begin_src emacs-lisp
(leaf package-name
  :require t
  :config
  (setq option-one 'value)
  (add-hook 'mode-hook 'function))
#+end_src

** Usage in Literate Configs
:PROPERTIES:
:tangle: no
:END:

When you write a package org file (like ~avy.org~), the literate-config-system automatically generates leaf forms from your code blocks:

#+begin_src org
* AVY
:PROPERTIES:
:PACKAGE: avy
:END:

** Initialization
:PROPERTIES:
:tangle: init
:END:

#+begin_src emacs-lisp
(require 'avy)
#+end_src

** Configuration
:PROPERTIES:
:tangle: config
:END:

#+begin_src emacs-lisp
(setq avy-keys '(?a ?s ?d ?f ?g ?h ?j ?k ?l))
#+end_src
#+end_src

The loader converts this to:

#+begin_src emacs-lisp
(leaf avy
  :require t
  :init (require 'avy)
  :config (setq avy-keys '(?a ?s ?d ?f ?g ?h ?j ?k ?l)))
#+end_src

** Guix Package Module
:PROPERTIES:
:tangle: no
:END:

Create ~/..emacs-packages/emacs-leaf.scm~:

#+begin_src scheme
(define-module (core-system user-space root editors emacs-packages emacs-leaf)
  #:use-module (gnu packages emacs-xyz)
  #:re-export (emacs-leaf))
#+end_src

** Architecture Notes
:PROPERTIES:
:tangle: no
:END:

Leaf fits perfectly into the literate-config-system because:

1. **Declarative** — One form per package, easy to understand
2. **Guix-compatible** — No external package managers needed
3. **Ordered loading** — `:after` keyword handles dependency ordering
4. **Error handling** — Errors don't crash the entire config
5. **Simple** — Minimal overhead, just a macro wrapper

The literate-config-system reads your org files, extracts code blocks, and automatically wraps the configuration in leaf forms for you.
