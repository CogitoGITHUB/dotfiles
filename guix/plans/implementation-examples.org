** Example 1: Simple Package - nushell

*** Source (guixos/gnu/packages/shells.scm)

What exists in =sources/guix/gnu/packages/shells.scm=:

#+begin_src scheme
(define-module (gnu packages shells)
  #:use-module (guix packages)
  #:use-module (guix build-system cargo)
  #:use-module (guix records)
  #:use-module (guix utils)
  #:use-module ((guix licenses) #:prefix license:)
  #:use-module (gnu packages compression)
  #:use-module (gnu packages pkg-config)
  #:use-module (gnu packages rust)
  #:use-module (gnu packages tls)
  ...
  #:export (nushell ...))

(define-public nushell
  (package
    (name "nushell")
    (version "0.96.1")
    (source
      (origin
        (method url-fetch)
        (uri (crate-uri "nu" version))
        (sha256
          (base32 "some-hash-here"))))
    (build-system cargo-build-system)
    (inputs
      (list openssl
            pkg-config))
    (synopsis "A new type of shell")
    (description "...")
    (home-page "https://www.nushell.sh")
    (license license:mit)))
#+end_src

*** Step 1: Create dependencies/nushell.scm

Extract ONLY the nushell package definition and update module name:

#+begin_src scheme
(define-module (dependencies nushell)
  #:use-module (guix packages)
  #:use-module (guix build-system cargo)
  #:use-module (guix records)
  #:use-module (guix utils)
  #:use-module ((guix licenses) #:prefix license:)
  #:use-module (gnu packages compression)
  #:use-module (gnu packages pkg-config)
  #:use-module (gnu packages rust)
  #:use-module (gnu packages tls)
  #:export (nushell))

(define-public nushell
  (package
    (name "nushell")
    (version "0.96.1")
    (source
      (origin
        (method url-fetch)
        (uri (crate-uri "nu" version))
        (sha256
          (base32 "some-hash-here"))))
    (build-system cargo-build-system)
    (inputs
      (list openssl
            pkg-config))
    (synopsis "A new type of shell")
    (description "...")
    (home-page "https://www.nushell.sh")
    (license license:mit)))
#+end_src

*Key changes:*
- Module name: =(gnu packages shells)= → =(dependencies nushell)=
- Export: =#:export (nushell ...)= → =#:export (nushell)=
- Everything else: UNCHANGED (including all imports from gnu packages)

*** Step 2: Update shell/nushell.scm

Current state:
#+begin_src scheme
(define-module (core-system user-space root shell nushell)
  #:use-module (gnu packages nushell)
  #:re-export (nushell))
#+end_src

⚠️  Note: This imports from =(gnu packages nushell)= which doesn't exist!
   The original definition is in =(gnu packages shells)=.
   After migration:

#+begin_src scheme
(define-module (core-system user-space root shell nushell)
  #:use-module (dependencies nushell)
  #:re-export (nushell))
#+end_src

*Key change:*
- Replace: =(gnu packages xyz)= → =(dependencies nushell)=
- Keep: =#:re-export (nushell)= UNCHANGED

*** Step 3: Verify with REPL

** Example 4: Package with Custom Dependency - hyprland

This is more complex because hyprland may depend on other custom packages in dependencies/

*** Scenario

Assume you have:
- =dependencies/hyprland.scm= (main package)
- =dependencies/wayland.scm= (dependency)
- =dependencies/libxkbcommon.scm= (dependency)

And hyprland's original definition has:

#+begin_src scheme
(define-module (gnu packages wm)
  ...
  #:use-module (gnu packages wm)      ;; contains both hyprland and wayland utilities
  ...
  #:export (hyprland ...))

(define-public hyprland
  (package
    (name "hyprland")
    ...
    (inputs
      (list wayland
            libxkbcommon
            ...))))
#+end_src

*** Step 1: Create dependencies/hyprland.scm

When creating hyprland.scm, if it depends on wayland package defined in dependencies/:

#+begin_src scheme
(define-module (dependencies hyprland)
  #:use-module (guix packages)
  #:use-module (guix build-system cmake)
  #:use-module (guix download)
  #:use-module ((guix licenses) #:prefix license:)
  ;; GNU package inputs
  #:use-module (gnu packages freedesktop)
  #:use-module (gnu packages wm)
  ;; CUSTOM package if in dependencies
  #:use-module (dependencies libxkbcommon)
  #:export (hyprland))

(define-public hyprland
  (package
    (name "hyprland")
    ...
    (inputs
      (list (assoc-ref (@ (dependencies libxkbcommon) libxkbcommon))
            ...))))
#+end_src

Alternative approach (simpler):
- Keep input references to packages in =(gnu packages ...)=
- Only import from =(dependencies ...)= if the package IS in your dependencies/

*** Step 2: Update desktop/hyprland.scm

Unchanged pattern:
#+begin_src scheme
(define-module (core-system user-space root desktop hyprland)
  #:use-module (dependencies hyprland)
  #:re-export (hyprland))
#+end_src

** Example 5: Troubleshooting - Missing Imports

*** Problem Scenario

You extract =curl.scm= and test with REPL:

#+begin_src bash
guile -c "(use-modules (dependencies curl))" 2>&1
#+end_src

You get error:
#+begin_src
ERROR: In procedure read-record-header:
  Unbound variable: zlib
#+end_src

*** Diagnosis

The curl package definition uses ~zlib~ but the import is missing:

Look in original source:
#+begin_src scheme
(define-public curl
  (package
    ...
    (inputs (list zlib ...))))
#+end_src

But you didn't copy:
#+begin_src scheme
#:use-module (gnu packages compression)  ;; provides zlib
#+end_src

*** Solution

Add missing import to dependencies/curl.scm:

#+begin_src scheme
(define-module (dependencies curl)
  #:use-module (guix packages)
  #:use-module (guix build-system gnu)
  #:use-module (guix download)
  #:use-module ((guix licenses) #:prefix license:)
  #:use-module (gnu packages compression)  ;; ← ADD THIS
  #:use-module (gnu packages perl)
  #:export (curl))
#+end_src

Re-test:
#+begin_src bash
guile -c "(use-modules (dependencies curl))" && echo "OK"
#+end_src

  #:use-module (core-system user-space root shell fetch fastfetch)

  #:re-export (nushell bash starship fzf zoxide zellij carapace atuin superfile television rip-cli
               btop htop ncdu glances
               upower tlp acpi
               zip unzip 7zip unrar-free zstd xz
               neofetch fastfetch)
  #:export (root-shell-packages
            root-shell-system-monitor-packages
            root-shell-power-packages
            root-shell-archive-packages
            root-shell-fetch-packages))

(define root-shell-packages
  (list nushell bash starship fzf zoxide zellij carapace atuin superfile television rip-cli))

(define-public root-shell-system-monitor-packages
  (list btop htop ncdu glances))

(define-public root-shell-power-packages
  (list upower tlp acpi))

(define-public root-shell-archive-packages
  (list zip unzip 7zip unrar-free zstd xz))

(define-public root-shell-fetch-packages
  (list neofetch fastfetch))
#+end_src

⚠️  IMPORTANT: Loaders DO NOT CHANGE during migration!
   The changes only happen in the imported modules (shell/*.scm files).
