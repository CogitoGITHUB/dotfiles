* LiterativeOS Guix Refactoring Plan
** Overview

Migrate from direct GNU package imports to a custom ~dependencies/~ directory structure that holds
extracted package definitions from ~sources/guix/~. This decouples your system config from upstream
Guix changes and allows selective package overrides.

** Current Architecture vs. Target Architecture

*** Current State
- Package modules: 3-5 lines, import from ~(gnu packages category)~ and re-export
  - Example: =shell/nushell.scm= imports ~(gnu packages nushell)~ and re-exports ~nushell~
- Loaders: Aggregate multiple packages and re-export for system composition
  - Example: =loaders/shell.scm= imports from all shell/* modules
- Direct coupling: System depends on upstream Guix module naming

*** Target State
- =dependencies/= flat directory contains full package definitions (one file per package)
  - Example: =dependencies/nushell.scm= has complete package definition + all imports
- Load path: ~guixos.scm~ adds ~/home/aoeu/.config/guix/dependencies~ to load path
- Package modules: 3-5 lines, import from ~(dependencies package-name)~ and re-export
  - Example: =shell/nushell.scm~ imports from ~(dependencies nushell)~ instead of ~(gnu packages nushell)~
- Loaders: Unchanged, still aggregate and re-export (no modifications needed)
- Decoupled: System uses custom package definitions, can override/patch freely

*** Phase 2: Extract Packages from sources/guix/
   :PROPERTIES:
   :EFFORT:   Varies (depends on number of packages)
   :STATUS:   pending
   :SUBTASKS: 28 category loaders to process
   :END:

For each package used in the categorized structure:
1. Find package definition in ~sources/guix/gnu/packages/*~ or ~sources/guix/guix/packages/*~
2. Extract full module definition (with all ~#:use-module~ imports)
3. Create new file in ~dependencies/package-name.scm~
4. Update module name from ~(gnu packages category)~ to ~(dependencies package-name)~
5. Export the package-public-name

*Example workflow for nushell:*

- Source file: =/home/aoeu/.config/guix/sources/guix/gnu/packages/shells.scm=
- Extract: The full ~(define-public nushell ...)~ definition + all required imports
- Create: =/home/aoeu/.config/guix/dependencies/nushell.scm=
- Content:
  #+begin_src scheme
  (define-module (dependencies nushell)
    #:use-module (guix packages)
    #:use-module (guix build-system cargo)
    #:use-module (gnu packages crates-io)
    #:use-module (gnu packages pkg-config)
    #:use-module (gnu packages tls)
    #:export (nushell))

  (define-public nushell
    (package ...))
  #+end_src

*Important notes:*
- Dependencies may require imports from ~(gnu packages ...)~ — leave these as-is
- Only extract ONE package per file (even if source file has multiple)
- Preserve all original imports, comments, and metadata
- If a package depends on another custom package, add ~#:use-module (dependencies other-package)~

*** Phase 4: Update Package Modules (category by category)
   :PROPERTIES:
   :EFFORT:   Varies (1-2 hours for all categories)
   :STATUS:   pending
   :END:

Update all 3-line package wrapper modules to import from dependencies instead of gnu packages.

*Example: shell/nushell.scm*

*Before:*

(define-module (core-system user-space root shell nushell)
  #:use-module (gnu packages nushell)
  #:re-export (nushell))
#+end_src

*After:*
#+begin_src scheme
(define-module (core-system user-space root shell nushell)
  #:use-module (dependencies nushell)
  #:re-export (nushell))
#+end_src

*Process for each category (e.g., shell):*
1. List all files in ~shell/~
2. For each ~shell/*.scm~ file:
   - Change ~#:use-module (gnu packages xyz)~ to ~#:use-module (dependencies xyz)~
   - If imports from multiple ~(gnu packages ...)~, consolidate to ~(dependencies ...)~ only
   - Keep ~#:re-export~ unchanged
3. Test with REPL: ~guile -c "(use-modules (core-system user-space root shell nushell)) (display nushell)"~

** Risks & Mitigation

| Risk | Mitigation |
|------|-----------|
| Missing dependencies in extracted packages | Verify all ~#:use-module~ lines before extraction |
| Packages depend on patches from gnu | Check for patches in original definitions; copy to dependencies/ if needed |
| Circular imports between custom packages | Use REPL to test each module after creation |
| Load order issues | Add dependencies/ first in load-path (before core-system) |
| Large number of files (1336 total in sources/) | Process by category; not all packages are used by your config |

** Rationale for This Approach

1. **Decoupling**: System no longer depends on upstream Guix package locations
2. **Selective overrides**: Easy to patch packages without modifying guix source
3. **Version stability**: Pin specific package versions in your config
4. **Custom packages**: Add new packages to dependencies/ without contributing upstream
5. **Gradual migration**: Can deploy category-by-category, test between each
