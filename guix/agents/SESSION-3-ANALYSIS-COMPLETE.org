## Key Themes to Understand

These are the main new concepts that documentation is missing:

### 1. Home vs System Services
```
BEFORE: Documentation only mentions system services (sudo herd)
AFTER: Clearly distinguish home services (herd) vs system (sudo herd)
WHY: Users trying "sudo herd emacs" won't find the home service
```

### 2. Environment Variables Matter
```
BEFORE: Environment variables barely mentioned
AFTER: Explicitly document XDG_RUNTIME_DIR, native compilation vars
WHY: Without these, daemon startup fails with cryptic errors
```

### 3. Home Reconfiguration is Different
```
BEFORE: Only "sudo guix system reconfigure" documented
AFTER: Explain "guix home reconfigure" for user packages/services
WHY: Users adding home packages won't activate them without knowing this
```

### 4. Module Paths Have Subdirectories
```
BEFORE: Module paths only mention (core-system user-space root ...)
AFTER: Explain both root/ and home/ subdirectories with examples
WHY: Users creating home services need to know the correct paths
```
