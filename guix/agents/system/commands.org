* Guix REPL (PRIMARY!)

Primary tool for all interactive work:

#+BEGIN_SRC bash
# Standard REPL
guix repl

# With development environment
guix shell -D -f /home/aoeu/.config/guix/config.scm -- guix repl
#+END_SRC

** Key REPL Commands

| Command | Description |
|---------|-------------|
| =,build <pkg>= | Build package, return path |
| =,phases <pkg>= | Show build phases |
| =,lower <obj>= | Lower to derivation |
| =,help guix= | All commands |

* DECLARATIVE SYSTEM (CRITICAL!)

This Guix system is DECLARATIVE. EVERYTHING must be in config.scm!

The entire system rebuilds with:
  =sudo guix system reconfigure ~/.config/guix/config.scm=

If it exists on the system but is NOT declared, it's not reproducible!

** What must be declared:**

- ALL packages (home/packages/*)
- ALL services (root/services/*)
- ALL kernel config (kernel/*)
- ALL users/permissions
- ALL configurations

If you add something to the system, it MUST be declared in Guix!

* Package + Service Organization (CRITICAL!)

Each package file must include its related service definition!

** Structure:
- home/packages/utils/keyd.scm → package + shepherd service
- home/packages/utils/kanata.scm → package + shepherd service
- home/packages/desktop/hypridle.scm → package + config

** root/services/ is ONLY for:
- Services requiring root privileges (openssh, tailscale, docker, greetd)

* CRITICAL: NO SUDO FOR READING FILES!

ALL files in the system are world-readable. NEVER use sudo to read files!

Common locations:
- /gnu/store/* - world-readable
- /tmp/* - world-readable
- /etc/* - world-readable
- /var/log/* - world-readable

ONLY use sudo for:
- Writing/modifying system files
- Service management (herd, systemctl)
- System operations (reboot, mount)
- Build and test with REPL
- This prevents the same problem twice

* System Build (ALWAYS USE THIS!)

#+BEGIN_SRC bash
gr
#+END_SRC

Works from any directory. Rebuilds, commits, and pushes on success.

** What the script does:
1. Runs =sudo guix system reconfigure=
2. Checks exit code - fails if non-zero
3. Scans output for "error:" - fails if found
4. Scans output for "warning:" - warns if found
5. Sends ntfy notification on completion
6. If SUCCESS: asks "Git push changes?"
7. If errors OR warnings: EXIT IMMEDIATELY, fix before continuing!

* Package Testing Workflow (ALWAYS FOLLOW THIS!)

IMPORTANT: After creating/modifying a package, ALWAYS test in this order:

1. *Create* the .scm file in =home/packages/=
2. *Test* with Guix REPL: =guix repl= then =,load "path.scm"= then =,build <pkg>=
3. *Debug* if needed: =guix build <pkg> -K= to keep failed build
4. *Test* the package: =guix package -i <name>= then =<name> --version=
5. *Build* system (only if package works): =sudo guix system reconfigure config.scm=
6. *Ask user*: "Rebuild system now?"
7. *After system works*: Ask "Git push changes?"

This saves huge time/energy - never skip to reconfigure without testing package first!
