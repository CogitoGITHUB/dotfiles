** Service not starting — home-environment services vs system services

Home-environment services (in ~/.config/guix/home/) are managed by USER shepherd, NOT root.
Use `herd` WITHOUT sudo:

# Home services (emacs daemon, wireplumber, etc)

herd log emacs             ; no sudo
herd restart emacs         ; no sudo
#+END_SRC

System services use sudo (network, ssh, etc). Never use sudo with home services.

** Service not starting — emacs daemon specific

The emacs daemon requires several environment variables to create socket correctly:

#+BEGIN_SRC sh
# Check if daemon is running
herd status emacs

# Check logs for errors
herd log emacs | tail -20

# Restart if not running
herd restart emacs
#+END_SRC

Common issues:
- Socket missing at ~/.emacs.d/server — check XDG_RUNTIME_DIR is set
- Error: "Cannot create socket" — verify /run/user/{uid} exists and is writable
- Error: "native compilation" — daemon.scm must have EMACS_NATIVECOMP_AOTCOMP=no

See also: [[../../Operating-System/core-system/user-space/home/emacs/daemon.scm][daemon.scm]]

#+BEGIN_SRC org
** Home-environment services vs system services

NOT ALL SERVICES ARE SYSTEM-LEVEL. Understand the difference:

*System services* (use shepherd-root-service-type, in Operating-System):
- Run as root
- Start at boot
- Accessible via `sudo herd status`
- Examples: sshd, nginx, user accounts
- Location: Operating-System/core-system/user-space/root/*/
- Use: (simple-service 'name shepherd-root-service-type ...)

*Home-environment services* (use home-shepherd-service-type, in home-environment):
- Run as current user
- Start when home-environment is activated
- Accessible via `herd status` (no sudo)
- Examples: emacs daemon, wireplumber, user daemons
- Location: Operating-System/core-system/user-space/home/*/
- Use: (simple-service 'name home-shepherd-service-type ...)

Current home services:
- emacs daemon (Operating-System/core-system/user-space/home/emacs/daemon.scm)
- PipeWire services (check home loaders)

If you create a new user-level daemon, use home-shepherd-service-type.

** Home env-vars don't reach shepherd
home-environment-variables-service-type only sets shell session vars (.bashrc, etc).
Shepherd services inherit NONE of them.

CRITICAL: Pass env vars directly in make-forkexec-constructor:
#+BEGIN_SRC scheme
(make-forkexec-constructor
  (list #$(file-append program "/bin/name"))
  #:environment-variables
  (list (string-append "XDG_RUNTIME_DIR=/run/user/" (number->string (getuid)))
        "OTHER_VAR=value"))
#+END_SRC

Do NOT rely on home-environment-variables-service-type for daemon config.
#+END_SRC

#+BEGIN_SRC org
** Emacs daemon service (shepherd service pattern)

Home-environment services use a different pattern than packages. Here's the working pattern
for emacs daemon (can adapt for other user-level daemons):

#+BEGIN_SRC scheme
(define-module (core-system user-space home emacs daemon)
  #:use-module (gnu home services)
  #:use-module (gnu home services shepherd)
  #:use-module (gnu packages emacs)
  #:use-module (guix gexp)
  #:export (home-emacs-daemon-service))

(define-public home-emacs-daemon-service
  (simple-service 'emacs-daemon
                  home-shepherd-service-type
                  (list
                    (shepherd-service
                      (provision '(emacs))
                      (documentation "Start the Emacs daemon for use with emacsclient.")
                      (requirement '())
                      (start #~(make-forkexec-constructor
                                 (list #$(file-append emacs "/bin/emacs")
                                       "--daemon")
                                 #:environment-variables
                                 (list (string-append "HOME=" (getenv "HOME"))
                                       (string-append "XDG_RUNTIME_DIR=/run/user/" 
                                                      (number->string (getuid)))
                                       "EMACS_NATIVECOMP_AOTCOMP=no"
                                       "EMACS_NATIVECOMP_DRIVER=no")
                                 #:log-file (string-append (getenv "HOME") "/.emacs.d/daemon.log")
                                 #:directory (getenv "HOME")))
                      (stop #~(make-kill-destructor))
                      (respawn? #f)
                      (one-shot? #f)))))
#+END_SRC

Key differences from system services:
1. Module path: (core-system user-space home ...) NOT (core-system user-space root ...)
2. Service type: home-shepherd-service-type NOT shepherd-root-service-type
3. #:environment-variables: CRITICAL for proper socket creation and daemon startup
4. Export: use #:export (not #:re-export) for the service definition
5. Logging: use user's ~/.emacs.d/daemon.log, NOT /var/log/

Critical environment variables for emacs daemon:
- XDG_RUNTIME_DIR — where socket is created (must use getuid, never hardcode)
- EMACS_NATIVECOMP_AOTCOMP=no — prevents native compilation failures
- EMACS_NATIVECOMP_DRIVER=no — prevents compiler driver loading at startup

See: /home/aoeu/.config/guix/Operating-System/core-system/user-space/home/emacs/daemon.scm
#+END_SRC
