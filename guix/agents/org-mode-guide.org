* Org Mode Guide

This guide covers the basics of Org mode for writing documentation.

** Headers

Use =*= for headers. More =*= = means deeper level:

#+BEGIN_SRC org
* Top level
** Second level
*** Third level
#+END_SRC

** Bold and Italic

#+BEGIN_SRC org
*bold*
/italic/
=code=
#+END_SRC

Renders as: *bold*, /italic/, =code=

** Lists

*** Unordered list
#+BEGIN_SRC org
- Item one
- Item two
  - Nested item
#+END_SRC

*** Ordered list
#+BEGIN_SRC org
1. First item
2. Second item
#+END_SRC

** Links

Internal link to another file:
#+BEGIN_SRC org
[[./filename.org][Link text]]
#+END_SRC

Link to a header in same file:
#+BEGIN_SRC org
[[*Header Name][Link text]]
#+END_SRC

** Code Blocks

#+BEGIN_SRC org
,#+BEGIN_SRC sh
echo "Hello"
,#+END_SRC
#+END_SRC

Shows as:
#+BEGIN_SRC sh
echo "Hello"
#+END_SRC

** Tables

Simple table:
#+BEGIN_SRC org
| Column 1 | Column 2 |
|----------|----------|
| Value 1  | Value 2  |
#+END_SRC

** Checkboxes

#+BEGIN_SRC org
- [ ] Unchecked task
- [X] Checked task
#+END_SRC

** Source Code Comments

=,#+BEGIN_SRC= and =,#+END_SRC= with comma prefix prevents execution when editing in Emacs.

The comma is only needed when writing the org file itself.
