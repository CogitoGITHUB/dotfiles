* LiterativeOS — Architecture: Notes & Gotchas

** !! CRITICAL RULE: Custom Modules for EVERY external package !!

If any package (like gzip, zlib, libxml2, etc.) is used as an input in a custom package definition, you MUST create a dedicated module for it:
  - Path: Operating-System/core-system/user-space/root/<category>/<name>.scm
  - Template: Full define-public with origin, uri, sha256 hash from `guix download`
  - THEN import that module in the package that needs it

NEVER import directly from (gnu packages ...) for these. Self-contained always.
This rule exists because Guix will be cut from the system.

** PipeWire/WirePlumber Must Live In Home Shepherd
- System shepherd (PID 1) cannot create proper user sessions
- WirePlumber needs elogind session context to detect ALSA/Bluetooth
- home-pipewire-service-type handles this correctly via home shepherd
- guix-home-service-type embeds home config in system reconfigure:
  (service guix-home-service-type (list (list "username" home-environment)))
- Module: (gnu services guix)
- After this, sudo guix system reconfigure deploys both system AND home

** Home vs System Audio Split
System shepherd: alsa-service (unmute), mpd-service
Home shepherd:   pipewire, wireplumber, pipewire-pulseaudio, dbus
