* Adding Packages and Services

** Getting the Correct Hash

Use =guix download= to get the hash - it returns the correct format automatically:

#+BEGIN_SRC sh
guix download https://github.com/author/repo/releases/download/v1.0.0/package.tar.gz
#+END_SRC

This returns a hash like =0nlcy211lrvj7vyl6zrd1v3hv3s264m0pd69xc7c3df6rc41ywsv= that you can use directly with =(base32 "...")=

*** Common Errors
- Don't use =sha256sum | base32= - it produces invalid base32
- Don't use =guix hash= - it returns nix-store format
- Use =guix download= for the correct hash format

** Packages and Services Stay in the Same File

In this setup, packages and services are defined together in each loader file. For example, =loaders/containers.scm= has both =root-containers-packages= and =root-containers-services=.

** Adding a New Submodule

If you want to add a new tool, create a new submodule file in the category directory, then import it in the loader.

Example: Adding =starship= to =shell/=

1. Create =shell/starship.scm=:
#+BEGIN_SRC scheme
(define-module (core-system user-space root shell starship)
  #:use-module (gnu packages rust)
  #:export (starship))

(define-public starship starship)
#+END_SRC

2. Update =loaders/shell.scm= to import it:
#+BEGIN_SRC scheme
(define-module (core-system user-space root loaders shell)
  #:use-module (core-system user-space root shell nushell)
  #:use-module (core-system user-space root shell starship)  ; <-- add
  #:re-export (nushell starship)
  #:export (root-shell-packages))

(define root-shell-packages
  (list nushell starship))  ; <-- add to list
#+END_SRC

** Adding a Package to Existing Loader

Example: Adding a package to =loaders/core.scm=:
#+BEGIN_SRC scheme
(define-public root-core-packages
  (list coreutils
        findutils
        grep
        ;; ... existing packages ...
        your-new-package))
#+END_SRC

** Adding a Service

Example: Adding a service to =loaders/containers.scm=:
#+BEGIN_SRC scheme
(define-public root-containers-services
  (list (service containerd-service-type)
        (service docker-service-type)
        (service your-new-service-type)))
#+END_SRC

** Finding Package Names

#+BEGIN_SRC sh
guix search package-name
guix package --show-package=package-name
#+END_SRC
