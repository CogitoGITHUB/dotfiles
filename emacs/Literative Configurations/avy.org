** Overview
:PROPERTIES:
:tangle: no
:END:

Avy is a simple, elegant jump navigation system for Emacs. Rather than moving the cursor slowly with arrow keys, avy lets you jump directly to any visible character using a decision tree of key presses.

*Key Features:*
- Jump to any visible character by prefix
- Multiple jump modes (char, word, line)
- Integration with god-mode and other modal editors
- Customizable key maps and styling

** Configuration
:PROPERTIES:
:tangle: config
:END:

#+begin_src emacs-lisp
;; ════════════════════════════════════════════════════════════
;; § AVY — CONFIGURATION
;; ════════════════════════════════════════════════════════════

;; Use home row keys for avy jumps
(setq avy-keys '(?a ?s ?d ?f ?g ?h ?j ?k ?l))

;; Highlight first decision char with a custom face
(setq avy-highlight-first t)

;; Use a gray background for avy overlays
(setq avy-background t)

;; All windows mode by default
(setq avy-all-windows t)

;; Case sensitivity
(setq avy-case-fold-search t)

;; Timeout for char input (in seconds)
(setq avy-timeout-seconds 0.5)

;; Style of avy overlays
(setq avy-style 'at-full)

#+end_src

** Keybindings
:PROPERTIES:
:tangle: config
:END:

#+begin_src emacs-lisp
;; ════════════════════════════════════════════════════════════
;; § AVY — KEYBINDINGS
;; ════════════════════════════════════════════════════════════

;; Jump to char
(global-set-key (kbd "C-:") #'avy-goto-char)
(global-set-key (kbd "C-'") #'avy-goto-char-2)

;; Jump to line
(global-set-key (kbd "M-g f") #'avy-goto-line)
(global-set-key (kbd "M-g w") #'avy-goto-word-1)

;; Jump to word beginning
(global-set-key (kbd "C-c j w") #'avy-goto-word-0)
(global-set-key (kbd "C-c j c") #'avy-goto-char)

;; Jump and copy region
(global-set-key (kbd "C-c j y") #'avy-copy-region)
(global-set-key (kbd "C-c j k") #'avy-kill-region)
(global-set-key (kbd "C-c j m") #'avy-move-region)

;; Avy integration with god-mode
(with-eval-after-load 'god-mode
  (define-key god-local-mode-map (kbd "j") #'avy-goto-char)
  (define-key god-local-mode-map (kbd "J") #'avy-goto-char-2))

#+end_src

** Guix Package Module
:PROPERTIES:
:tangle: no
:END:

Create or update ~/..emacs-packages/avy.scm~:

#+begin_src scheme
(define-module (core-system user-space root editors emacs-packages avy)
  #:use-module (gnu packages emacs-xyz)
  #:use-module (core-system user-space root editors emacs-packages literate-config-system)
  #:re-export (emacs-avy)
  #:export (avy-config))

;; Configuration metadata from avy.org
(define avy-config
  (make-lcs-config
    "avy.org"                    ; org-file: path to source .org
    "0.5.0"                      ; version
    '()                          ; depends: empty for now
    '()                          ; after: load after these packages
    #f                           ; built-in: not a built-in package
    #f))                         ; defer: load immediately

#+end_src

** Architecture Notes
:PROPERTIES:
:tangle: no
:END:

The new system workflow:

1. *Write* your configuration in this .org file with ~:tangle:~ blocks
2. *Define* a Guix module (~avy.scm~) that:
   - Re-exports the upstream package (~emacs-avy~)
   - Creates a metadata record pointing to this .org file
   - Includes any custom dependencies or ordering
3. *Bootstrap* Emacs by calling ~literate-config-load~
4. The loader will:
   - Read this .org file from ~/Literative Configurations/avy.org~
   - Extract init blocks and config blocks
   - Build a ~leaf~ form with your settings
   - Evaluate it using the Guix-provided package

No need for straight, no package.el conflict—just Guix packages + leaf.el + literate org files.
