* Org-Supertag
:PROPERTIES:
:PACKAGE: org-supertag
:CATEGORY: system
:END:

Hierarchical tagging system for Org-mode with powerful query and filtering capabilities.

** Initialization
:PROPERTIES:
:tangle: init
:END:

#+BEGIN_SRC emacs-lisp
(add-to-list 'load-path 
             (expand-file-name "lisp/org-supertag" user-emacs-directory))
#+END_SRC

** Configuration
:PROPERTIES:
:tangle: config
:END:

#+BEGIN_SRC emacs-lisp
(setq org-supertag-sync-directories '("~/.config/emacs"
                                      "~/org"
                                      "~/.config"))
(when (fboundp 'supertag-vault-indicator-mode)
  (supertag-vault-indicator-mode 1))
#+END_SRC

** LiterativeOS Package Tags

LiterativeOS integrates org-supertag with literate-config-system to automatically tag and organize packages:

*** Tag Categories

The following categories organize LiterativeOS packages:

- *ai* - AI and machine learning packages (opencode, mcp, org-mcp, emacs-org-mcp)
- *development* - Development and REPL tools (geiser)
- *editing* - Text editing enhancements (god-mode, multiple-cursors)
- *framework* - Core frameworks and loaders (leaf)
- *navigation* - Text navigation packages (avy)
- *system* - System management packages (org-supertag)
- *ui* - User interface packages (modus-themes, dashboard)

*** Package Registry

When literate-config-system loads packages, it automatically:

1. Creates nodes for each package in org-supertag
2. Tags each node with its category from the :CATEGORY: property
3. Stores package metadata (filename, category) for queries
4. Enables filtering packages by category

*** Querying Package Tags

List all packages in a category:

#+BEGIN_SRC emacs-lisp
;; Find all AI packages
(let ((ai-packages (mapcar (lambda (meta) (car meta))
                           (seq-filter (lambda (m) (equal (nth 2 m) "ai"))
                                       lc--package-metadata))))
  ai-packages)
;; → ("opencode" "mcp" "emacs-org-mcp" "org-mcp")

;; Find all editing packages
(let ((editing-packages (mapcar (lambda (meta) (car meta))
                                (seq-filter (lambda (m) (equal (nth 2 m) "editing"))
                                            lc--package-metadata))))
  editing-packages)
;; → ("god-mode" "multiple-cursors")
#+END_SRC
